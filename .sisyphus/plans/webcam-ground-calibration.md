# Webcam + Ground Plane Calibration System

## TL;DR

> **Quick Summary**: Complete system for detecting thumbs-up gestures via webcam and mapping hand position to floor coordinates using perspective calibration.
> > **Components**:
> 1. **Calibration Tool**: Click 4 floor corners + enter dimensions → save homography matrix
> 2. **Detection Runtime**: Webcam capture, YOLO inference, FPS display
> 3. **Ground Mapping**: Transform hand bottom-center to ground coordinates
> 4. **Bird's Eye View**: Real-time top-down visualization
> > **Estimated Effort**: Medium (~2-3 hours)
> **Parallel Execution**: YES - 2 waves (calibration + detection foundation in parallel, then integration)
> **Critical Path**: Task 1 (calibration tool) + Task 2 (webcam base) → Task 5 (integration) → Task 6 (bird's eye view)

---

## Context

### Original Request
User trained a YOLO thumbs-up detection model on Google Colab and wants to run it on local webcam.

### Scope Evolution
1. **Initial**: Add webcam support to existing inference script
2. **Expanded**: Add ground plane calibration to map hand position to floor coordinates
3. **Final Use Case**: "Drop a plumb line from hand to floor - where does it hit?"

### Interview Summary
**Key Decisions**:
- Webcam: Index 0 (default), mirrored display
- Confidence threshold: 0.25
- Floor visible: Yes, in camera view
- Output: Pixels on ground image (bird's eye view coordinates)
- Projection: Direct downward (not pointing direction)

### Research Findings
**Calibration Approach**: Planar Homography using OpenCV
- `cv2.getPerspectiveTransform()` from 4 clicked points
- `cv2.perspectiveTransform()` to map detections to ground
- `cv2.warpPerspective()` to generate bird's eye view
- `cv2.FileStorage` to save/load calibration matrix

**Critical Constraint**: Homography assumes Z=0 (points ON ground). Raised hands introduce error (appear further from camera). Acceptable for approximate positioning.

### Core Objective
Build a complete system with: (1) calibration tool for ground plane definition, (2) webcam-based hand detection, (3) real-time mapping of hand position to ground coordinates, (4) bird's eye view visualization.

### Concrete Deliverables
- `calibrate_ground.py` - GUI tool for calibration (click 4 points + enter dimensions)
- Modified `inference.py` - Webcam support + ground mapping + bird's eye view
- `calibration.yml` - Saved homography matrix (generated by calibration tool)
- Documentation for calibration procedure

### Definition of Done
- [ ] Calibration tool successfully creates valid `calibration.yml`
- [ ] Running inference opens 2 windows: (1) main webcam view, (2) bird's eye view
- [ ] Hand detection works on webcam with FPS display
- [ ] Hand position shown as dot on bird's eye view
- [ ] Ground coordinates (x, y in pixels) displayed on screen
- [ ] Press 'q' gracefully exits all windows

### Must Have
- Calibration: Click 4 corners of rectangle on floor, enter real-world dimensions
- Calibration storage: YAML file with homography matrix
- Webcam: Index 0, mirrored display, FPS counter, 0.25 confidence
- Coordinate mapping: Bottom-center of bounding box → ground plane
- Bird's eye view: Real-time warped view showing hand position
- Video output: Save main view with detections overlaid

### Must NOT Have (Guardrails)
- No lens distortion correction (assumes acceptable error for hand height)
- No extrapolation outside calibrated region
- No pointing direction calculation (only vertical projection)
- No changes to model weights or training
